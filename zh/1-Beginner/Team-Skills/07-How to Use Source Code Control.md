# 如何使用原始碼控制
[//]: # (Version:1.0.0)
原始碼控制系統（又稱版本控制系統）讓你高效地管理工程。他們對一個人是很有用的，對一個團隊是至關重要的。它們追蹤不同版本里的所有改變，以至於所有程式碼都未曾丟失，其含義可以歸屬於改變。有了原始碼控制系統，一個人可以自信地寫一些而半途而廢的程式碼和偵錯的程式碼，因為你修改的程式碼被仔細地與提交的、官方的即將與團隊共享或釋出的程式碼分割開。

我挺晚才開始意識到原始碼控制系統的好處，但現在即使是一個人的工程，我也不能離開原始碼控制系統。當你們團隊在同樣的程式碼基礎上工作時，通常它們是必要的。然而，它們有另一個巨大的優點：它們鼓勵我們把程式碼當做一個成長的有機系統。因為每個改變都會被標記為帶有名字或數字的修正，一個人會開始認為軟體是一種可見的一系列漸進的提升。我認為這對初學者是尤其有用的。

使用原始碼控制系統的一個好的技術是一直保持在幾天後提交更新。在提交後，一定程度上不活躍，不被呼叫的程式碼在幾天內都不會完成，因此也不會對其他任何人產生任何問題。因提交錯誤的程式碼而降低你隊友的開發速度是一個嚴重的錯誤，這往往是一種禁忌。
